Definitions

  • origin: Alias for repository (e.g., https://github.com/shealy2020/MVC.git).

Helpful Git Commands

  • git remote -v //shows all origin alias commands
  • git branch [Branch Name] //Create Branch
  • git push -u origin [Branch Name] //Push New Branch to Remote Server
  • git checkout develop //Switch to another local branch such as develop
  • git pull //Once switched (to develop), integrates any remote changes with local develop
  • git push //Update the remote develop with local develop, which I worked on, saved, staged, and committed.

Intention

I want to do the following when I need to make a changed to Git-controlled files:

See Arnold's commands below for Git workflow.

  • get the most up-to-date version from (origin/develop)
    • git status --branch //find out which local branch I’m on.
    • git checkout documentation //If I need to switch to documentation from another local branch.
    • git pull origin develop //pulls down remote develop changes to local, checked out branch (e.g., documentation). Pull command also tries to merge develop and documentation if changes in documentation exists.
  • edit file(s)
    • git status //status shows which files have been modified
  • stages file(s) in preparation for commit
    • git add [file] //stages file(s)
    • git add -A //adds all unstaged (and untracked) files
    • git add . //stages all files in current dir
  • commit change
    • git commit -m "This is a commit message!" //creates a local commit, which are not yet pushed to origin.
    • git commit -a -m "This is a commit message!" //commits all (-a) local changes w/o staging (git add) them.
  • ALT-1: create a pull request
    1. Go to https://ingeniuxdev.visualstudio.com/Ingeniux/
    2. From menu select Code > Pull Requests > New pull request.

      Update: Click plus icon (+). Choose New Pull request. New navigation is a factor of new toolbar setup.

    3. Change selections in Base-to-Target drop-down menus to the following: documentation > develop.

      Note: develop will be under the All branches tab.

    4. Enter a description, etc.
    5. Click Create.
  • ALT-2: push to remote
    1. git push origin develop //updates the remote documentation with the local develop
  • ALT-3: merge the documentation into develop
    1. git checkout develop
    2. git merge documentation
